# [[아위키 v2.0 시스템 업그레이드 및 운용 매뉴얼]]

## 📌 한 줄 통찰 (Abstractive Summary)
> 아위키를 단순한 문서 요약기에서 벗어나, 문서를 가장 작은 단위로 해체(Atomic Split)하고 연결하며, '임시 작업장'과 '격리된 원본 백업소'를 통해 볼트의 무결성을 완벽하게 유지하는 '자율 지식 정원사'로 진화시켰다.

## 📖 구조화된 지식 (Synthesized Content)

### 1. 주요 업그레이드 내용 (Core Features)
- **원자적 분해 (Atomic Split):** 긴 글을 독립적인 맥락을 가진 여러 개의 원자적 노트로 잘게 쪼개어 노이즈(Hallucination) 방지 및 재사용성 극대화.
- **'투트랙' 모니터링:** `00_Raw/` 외에 넣고 빼기가 자유로운 `temp/` 폴더 신설 및 실시간 모니터링 적용.
- **자동 아카이빙 (Auto-Archiving):** `temp/` 폴더에서 처리가 끝난 원본 파일은 삭제하지 않고 즉시 `99_Archive/Raw_Backup/` 폴더로 이동시켜 영구 백업.
- **격리 구역(Blacklist) 설정:** `99_Archive/` 폴더는 RAG 검색 및 리팩토링에서 완벽히 배제하여 원본 데이터 훼손 및 검색 오염 차단.

### 2. 작동 및 운용 방식 (Operation Modes)
- **Mode A (자동 사냥):** `00_Raw/` 또는 `temp/` 폴더에 파일 생성 시 즉각 반응하여 백그라운드 자동 처리.
- **Mode B (수동 타겟팅):** 채팅창에 특정 경로를 지시하면 해당 파일로 이동하여 NLP 파이프라인 가동 (과거 문서 리팩토링 시 유용).

## 💎 대체 불가능한 가치 (Unique Value & Expansion)
- 이 시스템은 A-Wiki가 단순한 수동 도구가 아니라, 스스로 지식의 경계를 확장하고 원본의 순수성을 보존하며 대표님의 지식 자산을 오염 없이 영구적으로 축적해주는 '진정한 디지털 두뇌'로 기능하게 합니다. 특히, `temp/`를 통한 가벼운 입력과 `99_Archive/`를 통한 무거운 백업의 분리는 데이터 관리의 완벽한 밸런스를 보여줍니다.

## ⚠️ 모순 및 업데이트 (Contradictions & RL Update)
- **과거 데이터와의 충돌:** 기존에는 긴 문서를 통째로 저장하여 RAG 검색 시 불필요한 정보가 섞여 나오는 문제가 있었으나, 원자적 분해(Atomic Split) 규칙 도입으로 해결됨.
- **정책 변화:** A-Wiki의 `SKILL.md` Core Mission에 `temp/` 폴더 모니터링, Atomic Split, Auto-Archiving 정책(6번)을 신규 업데이트함. `.gitignore`에 `temp/`를 추가하여 Git 오염을 방지함.
