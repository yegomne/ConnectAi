# [[아위키 파이썬 핫폴더 자동화 스크립트 가이드]]

## 📌 한 줄 통찰 (Abstractive Summary)
> 파이썬의 Watchdog 라이브러리를 통해 `temp` 폴더를 실시간 감시함으로써, 마크다운은 물론 **엑셀(.xlsx)과 워드(.docx)** 파일까지 던지는 즉시 마크다운으로 자동 파싱하여 A-Wiki 에이전트를 깨우는 "잡식성(Omnivore) 자율 지식 파이프라인"을 완성했습니다.

## 📖 구조화된 지식 (Synthesized Content)

### 실시간 감지 자동화 스크립트 (Python Watchdog)
- **목적:** 사용자가 파일을 `temp/` 폴더에 드래그 앤 드롭하는 즉시 딜레이 없이 Antigravity CLI를 호출하여 A-Wiki 에이전트의 NLP 파이프라인을 가동.
- **옴니보어(Omnivore) 자동 파싱 로직:** 
  1. `watchdog`이 `.md`, `.xlsx`, `.docx` 파일 생성을 감지.
  2. **엑셀/워드 처리:** `pandas`와 `python-docx` 라이브러리를 통해 데이터를 읽어 마크다운 테이블 및 텍스트로 추출한 뒤, `temp/` 폴더 내에 `.md` 파일로 재생성. (원본은 `99_Archive/Raw_Backup`으로 즉시 이동)
  3. **A-Wiki 트리거:** 새롭게 생성되거나 들어온 `.md` 파일은 다시 왓치독의 감지망에 걸려 즉시 `antigravity run A-Wiki` 명령어 실행.
- **안전 설정:** 스크립트 파일 자체는 `temp/` 폴더 밖인 **`D:\Connect_Ai\awiki_watcher.py`** 경로에 안전하게 위치함.

## 💎 대체 불가능한 가치 (Unique Value & Expansion)
- 이제 대표님은 엑셀이나 워드로 작성된 업무 데이터, 회의록, 정산 내역 등을 굳이 텍스트로 변환할 필요가 없습니다. 파일 원본을 그대로 `temp/`에 던지기만 하면, 왓치독이 스스로 씹어(Parse) 소화하기 좋은 상태(.md)로 만든 뒤, 아위키의 두뇌로 직행시킵니다. 데이터 포맷의 장벽이 완전히 붕괴된 궁극의 'Hands-free' 지식화입니다.

## ⚠️ 모순 및 업데이트 (Contradictions & RL Update)
- **과거 데이터와의 충돌:** 기존에는 오직 마크다운(`.md`) 파일만 처리할 수 있었으나, 파이썬 파서(Parser) 로직을 통합하여 바이너리 오피스 파일까지 수용 가능하도록 아키텍처가 업그레이드됨.
- **정책 변화:** `temp/` 폴더 내의 `.md` 외의 원본 파일(.xlsx, .docx)은 파싱 즉시 `99_Archive/Raw_Backup/`으로 강제 격리(Archiving)하는 규칙 추가.
