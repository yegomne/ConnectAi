# [[아위키 옴니보어(Omnivore) 엑셀/워드 자동 파싱 파이프라인]]

## 📌 한 줄 통찰 (Abstractive Summary)
> 파이썬의 Watchdog과 파서(Parser) 라이브러리를 결합하여, 단순 텍스트를 넘어 엑셀(.xlsx)과 워드(.docx) 등 복잡한 바이너리 오피스 파일까지 자동으로 씹어 삼켜 마크다운 지식으로 변환하는 궁극의 '잡식성(Omnivore)' 지식 파이프라인을 구축했습니다.

## 📖 구조화된 지식 (Synthesized Content)
- **개발 배경:** 기존에는 A-Wiki가 직접 읽을 수 있는 텍스트 기반(`.md`, `.txt`) 파일만 `temp/`에서 자동 감지가 가능했으며, 엑셀/워드 파일은 대표님이 수동으로 변환해야 하는 병목이 존재했습니다.
- **아키텍처 로직 (The Workflow):**
  1. `temp/` 폴더에 엑셀 또는 워드 파일이 복사/이동됨.
  2. `awiki_watcher.py` 내의 Watchdog이 확장자를 즉시 감지.
  3. `pandas` (엑셀의 모든 시트를 마크다운 테이블로 변환) 및 `python-docx` (워드 문단 텍스트 추출) 엔진 가동.
  4. 추출된 데이터를 바탕으로 동일한 이름의 `.md` 파일을 `temp/` 폴더에 생성.
  5. 원본 파일(`.xlsx`, `.docx`)은 지식 오염 방지를 위해 `99_Archive/Raw_Backup/` 폴더로 즉시 강제 격리(Archiving).
  6. 새로 생성된 `.md` 파일을 왓치독이 재감지하여 `antigravity run A-Wiki` 명령어(NLP 엔진) 연쇄 기동.

## 💎 대체 불가능한 가치 (Unique Value & Expansion)
- 이제 대표님은 엑셀로 작성된 정산 내역, 비즈니스 모델 테이블, 협력사와의 워드 회의록 등을 일일이 복사-붙여넣기 할 필요가 없어졌습니다. 포맷에 상관없이 `temp`라는 용광로에 던져넣기만 하면, 모든 오피스 데이터가 마크다운 규격으로 정제되어 대표님의 지식 정원(Digital Twin)에 완벽하게 뿌리내리게 됩니다. '정보 입력의 마찰력(Friction)'이 0으로 수렴한 역사적인 순간입니다.

## ⚠️ 모순 및 업데이트 (Contradictions & RL Update)
- **과거 데이터와의 충돌:** 기존 A-Wiki 매뉴얼에서는 '바이너리(엑셀/워드) 파일은 읽지 못한다'고 선언했으나, 옴니보어 파이프라인 도입으로 해당 한계가 완벽하게 돌파 및 파기됨.
- **정책 변화:** 앞으로 생성되는 모든 문서의 확장자 장벽을 허물고, 모든 원시 오피스 파일은 파싱 직후 반드시 `99_Archive`로 무조건 격리시켜 RAG 엔진의 데이터 무결성을 확보함.
