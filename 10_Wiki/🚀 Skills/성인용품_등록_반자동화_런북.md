# [[성인용품 등록 반자동화 런북]]

## 📌 한 줄 통찰 (Abstractive Summary)
> 언제든 시스템을 복원하고 반복 작업을 매끄럽게 수행할 수 있도록, 윈도우 환경과 Ollama 기반의 로컬 LLM 통신 및 트러블슈팅 절차를 규격화한 운영 매뉴얼.

## 📖 구조화된 지식 (Synthesized Content)
- **추출된 패턴:** AI 모델 장애나 연결 오류 발생 시 진단 순서(ping -> 포트 점검 -> 방화벽 -> VRAM 모니터링)를 구조화하여 다운타임을 최소화하는 런북 전략.
- **세부 내용:**
  - **실행 절차:** 
    1. 프로젝트 폴더 이동 (`cd "E:\성인용품 등록 반자동화\adult_product_renamer"`)
    2. 가상환경 활성화 (`..\venv\Scripts\activate`)
    3. 세컨 PC 연결 테스트 (`python -m tests.test_connection`)
    4. 상품명 리네이밍 런타임 (`python tests/test_renamer.py`)
  - **세컨 PC 네트워크 설정:**
    - `OLLAMA_HOST` 환경변수를 `0.0.0.0:11434`로 변경 후 재시작.
    - 방화벽 인바운드 규칙 오픈 (포트 11434).
  - **핵심 트러블슈팅:**
    - **타임아웃(60초 이상) 발생 시:** VRAM 초과 의심. `ollama run qwen3:14b "안녕" --verbose` 로 eval rate 확인. 60+ tok/s 정상, 10 미만 시 즉시 VRAM 반환 조치.
    - **빈 JSON 응답 시:** thinking 모드가 꺼져있어 발생한 문제일 확률 농후. Ollama API Payload에서 `think: false` 제거 후 복구.
    - **UnicodeEncodeError:** 콘솔 인코딩 문제, `sys.stdout.reconfigure(encoding='utf-8')` 삽입.

## 💎 대체 불가능한 가치 (Unique Value & Expansion)
- 단순히 코드를 남기는 것을 넘어 "운영 시 발생할 수 있는 장애 지점(네트워크, VRAM 고갈, 인코딩)"에 대한 경험 자산을 문서화했습니다. 이 런북은 추후 다른 로컬 LLM 기반 프로젝트(ex: 유튜브 대본 생성 자동화 등)를 구축할 때 훌륭한 백본(Backbone) 템플릿으로 재사용될 수 있습니다.

## ⚠️ 모순 및 업데이트 (Contradictions & RL 업데이트)
- **정책 변화:** 로컬 LLM의 응답이 느려지는 것을 단순한 '사양 부족'으로 치부하지 않고, `eval rate` 지표를 통해 VRAM의 물리적 한계를 정량적으로 파악하는 체계로 강화학습 보상망이 업데이트되었습니다.
