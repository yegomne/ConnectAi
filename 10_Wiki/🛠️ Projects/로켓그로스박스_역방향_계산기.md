# [[AI 역방향 박스 계산기 (로켓그로스박스)]]

## 📌 한 줄 통찰 (Abstractive Summary)
> 외부 3D 라이브러리 없이 자체 SVG 렌더러와 Extreme Point(EP) 휴리스틱 빈 패킹 알고리즘을 결합하여, 상품의 최적 적재 시뮬레이션 및 잔여 공간 '꽉 채우기'를 서버 없이 브라우저 단에서 구현해 낸 혁신적인 물류 최적화 SPA입니다.

## 📖 구조화된 지식 (Synthesized Content)
- **핵심 아키텍처:**
  - **프론트엔드:** React 18, Tailwind CSS, Babel (단일 HTML 파일 내 CDN 로드)
  - **렌더링:** Three.js 없이 SVG 기반 커스텀 3D 투영 알고리즘 자체 구현 (드래그/터치 인터랙션, 자동 회전, 오버플로우 경고).
- **알고리즘 고도화 (EP 빈 패킹):**
  - 기존 3방향 모서리 후보지 방식의 한계를 극복하기 위해 유부장이 **Extreme Point(EP) 3D 빈 패킹**을 도입. 크기가 다른 박스 혼적 시 발생하는 미세한 틈새(Gap) 좌표를 수학적으로 역산하여 정확히 채워 넣음.
- **실무 중심의 핵심 기능:**
  - **자동/수동 모드:** 사용자가 지정한 큰 박스 내 적재 시뮬레이션 및 상품 체적 기반 최적의 큰 박스 자동 추천.
  - **수량 최대화 (꽉 채우기):** `skipOnFail` 옵션을 통해 들어가지 않는 박스를 건너뛰고 남은 틈새까지 물리적 빈 공간이 없어질 때까지 욱여넣는 기능 (인디고색 UI 버튼).
  - **데이터 지속성:** `localStorage`를 활용한 프리셋(우체국 박스 규격 등) 및 마지막 UI 상태 유지.
- **트러블슈팅 (호이스팅 버그):**
  - `useEffect`가 `useState` 선언부보다 상단에 위치해 의존성 배열(deps)이 `undefined`로 고정되는 Babel 호이스팅 버그 발생. 
  - 이를 하단으로 이동시켜 `localStorage` 저장 불능(혼적 슬라이더 버그 등) 문제를 완벽히 해결함.

## 💎 대체 불가능한 가치 (Unique Value & Expansion)
- 이 솔루션은 단순히 박스를 계산하는 툴을 넘어, 값비싼 서버 비용과 무거운 3D 라이브러리 종속성 없이 브라우저 자체의 수학 연산(React + SVG)만으로 물류 최적화를 이뤄낸 '경량화 아키텍처의 표본'입니다. 이 코드 베이스의 EP 패킹 알고리즘과 상태 관리 패턴은 향후 물류뿐만 아니라 창고 관리(WMS), 컨테이너 적재 시뮬레이션 등 모든 공간 최적화 비즈니스로 확장할 수 있는 범용적이고 폭발적인 가치를 지닙니다.

## ⚠️ 모순 및 업데이트 (Contradictions & RL Update)
- **과거 데이터와의 충돌:** 초기 단순 부피 합산 방식에서, 물리적인 3D 렌더링 및 EP 알고리즘 기반의 실측(Real-world) 시뮬레이션 방식으로 계산 패러다임이 전면 수정(Update)됨.
- **정책 변화:** UI 상태 관리 시 발생했던 호이스팅(Hoisting) 이슈 사례를 학습하여, 향후 React 기반 단일 HTML 파일 개발 시 선언 순서(Declaration Order)에 대한 코드 리뷰 가중치를 상향(🧠).
