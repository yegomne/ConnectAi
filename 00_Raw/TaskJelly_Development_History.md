# TaskJelly 프로젝트 개발 히스토리 요약 (v1.0 Glassmorphism)

본 문서는 TaskJelly 프로젝트의 초기 생성부터 현재 백업까지의 개발 과정, 발생했던 문제점 및 해결 방법(실수 포함), 그리고 새롭게 생성되거나 수정된 핵심 요소들을 기록한 개발 로그(Post-mortem)입니다.

---

## 1. 🚀 신규 생성된 핵심 기능 및 컴포넌트 (Created)
- **Vite React 기반 초기 프로젝트 셋업**: `e:\TaskJelly\app` 폴더 내 프론트엔드 기본 구조 생성.
- **상태 관리소 (Zustand Stores)**:
  - `TimerStore.ts`: 포모도로 타이머의 핵심 로직(작업 시간, 남은 시간, 실행/일시정지/종료 등)을 관리.
  - `TaskStore.ts`: 사용자의 할 일 항목(Task)을 추가하고 관리하는 역할 수행.
- **JellyBar (메인 컴포넌트)**:
  - `JellyBar.tsx`: 알약 형태의 각 할 일을 시각적으로 보여주고, 클릭 시 상태(Idle -> Running -> Locked)를 전환하는 핵심 UI 컴포넌트.
  - **react-spring 애니메이션 적용**: 포커스 시 쫀쫀하게 숨 쉬는 듯한(breathing) 애니메이션과, 잠금 상태에서 클릭 시 부르르 떨리는(shake) 타이포/너비 반응형 애니메이션 부여.
- **글래스모피즘(Glassmorphism) 스타일링**:
  - `JellyBar.css` 생성. 빛 반사(`::after`), 3D 내부 그림자(`::before`) 효과를 적용한 입체 알약 바 구현.
  - 동적 무지개 파스텔 테마 배정: IDLE 상태의 Task들에 대해 6가지 고유한 크리스탈 파스텔 색상을 `taskId` 해시 기반으로 고정 할당하여, 렌더링 시 색상이 깜빡이거나 변하는 현상 방지.

## 2. 🛠️ 발생했던 실수 및 해결 과정 (Mistakes & Fixes)
- **실수 1: '유리(Glass)' 질감의 투명도 부족 현상**
  - **이슈**: 초기 글래스모피즘 알약 디자인 시, 색상의 기준 불투명도(Opacity)가 너무 높아 뒷배경이 비치지 않고 답답한 "솔리드 컬러" 플라스틱처럼 보임.
  - **해결(Fixed)**: 
    - `rgba()` 배경 불투명도를 `0.45 ~ 0.55`로 낮춤.
    - 뒤가 예쁘게 비치도록 `backdrop-filter: blur(18px) saturate(130%)`을 적용해 굴절된 배경의 색이 채도가 높게 투과되도록 개선.
    - 테두리를 `1.5px solid rgba(255, 255, 255, 0.6)`으로 굵고 선명하게 다듬어 두꺼운 실제 유리의 반사광 효과 극대화.
- **실수 2: 랜덤 색상 배정 시 렌더링마다 색이 바뀌는 '플리커링(Flickering)' 현상 우려**
  - **이슈**: 단순히 `Math.random()`을 사용하여 각 Task에 색상을 배정할 경우, 타이머가 틱(Tick)을 발생시키는 등 앱이 리렌더링 될 때마다 알약 바의 색상이 계속 바뀌어 눈을 피로하게 함.
  - **해결(Fixed)**: `useMemo`와 내부 문자열 해싱(Hashing) 알고리즘을 사용해, `taskId` 텍스트 값에 따라 0~5번 색상 인덱스를 결괏값으로 고정 계산함. (언제 렌더링 되어도 색상이 고정 유지되도록 구조적 안정화)

## 3. 🗑️ 삭제되거나 대체된 항목 (Deleted / Removed)
- **단일 Active Color 클래스 폐기**: 모든 일반 할 일 바에 공통으로 적용되던 단일 파란색 클래스(`glossy-pill-color-active`)를 삭제함.
- **기본 App 템플릿 제거**: Vite가 기본 제공하는 불필요한 보일러플레이트 코드, 기본 로고, 기본 CSS 파일을 걷어내고 TaskJelly만의 순수 투명한 포커스/백그라운드 UI로 리빌딩.

## 4. 📦 최종 백업 (Milestone)
- 해당 버전을 안전하게 보관하기 위해 무거운 `node_modules`를 제외한 클린 백업 파일(`TaskJelly_v1_Glassmorphism_Backup.zip`)을 생성하여 프로젝트 루트에 저장 완료.

## 5. 🖱️ 드래그 앤 드롭 (Drag & Drop) 기능 구현 (Added)
- **@dnd-kit 도입**: 사용자에게 가장 매끄럽고 물리적 직관성이 뛰어난 `@dnd-kit/core` 및 `@dnd-kit/sortable` 모듈을 적용해 할 일 간 순서 재정렬 기능 구현 완료.
- **포인터 클릭 충돌 해결 (UX)**: `JellyBar` 안을 클릭했을 때 타이머가 즉시 켜지는 기능과 드래그 기능이 충돌하지 않도록 `PointerSensor`에 `activationConstraint: { distance: 5 }`를 추가. 5픽셀 이상 움직여야만 드래그로 판정하도록 정교한 입력 분리 성공.
- **모바일 환경 드래그 처리**: CSS에 `touch-action: none;`을 명시하여 모바일 환경에서 드래그 앤 드롭 시 전체 웹 브라우저가 스크롤되는 불쾌한 버그를 완벽히 통제.

## 6. 🐛 타입스크립트(TypeScript) 엄격 모드 런타임 에러 디버깅 (Fixed)
- **이슈**: 사용하지 않는 모듈(`Play` 등) import, 느슨한 타입(`any`), 그리고 상태 스토어 인터페이스의 시그니처 누락(`updateDailyGauge` 등)으로 인해, Vite 빌드 에러 및 브라우저 빈 화면(White Screen) 렌더링 중단 사태 발생.
- **해결(Fixed)**: 
  - 타임아웃 이벤트에 React 호환 타입 지정(`ReturnType<typeof setTimeout>`).
  - `import type` 문법 분리 적용 및 모든 암묵적 `any` 변수 제거.
  - 디버깅 완료 후 `npm run build` 에러 잔존율 0% 달성 및 로컬 포트 정상 서빙 확인.

## 7. 🎉 성취 보상 시스템: Daily Gauge 및 폭죽 모션 연동 (Added)
- **도파민 게이지 애니메이션 (`DailyGauge.tsx`)**: Task 완료(`TimerStore`의 `finish`) 시 10점씩 상승하며, `@react-spring/web`을 활용해 게이지 바가 차오르고 숫자가 카운팅되는 형태의 시각적 피드백 제공.
- **폭죽 파티클 (`canvas-confetti`)**: 게이지 점수가 이전보다 상승할 경우, 화면 하단에서 파스텔 톤 5색상 폭죽이 팡팡 터져 시각적 쾌감을 줌.
- **Memory 상태 필터링**: 작업을 완료한 Task는 상태가 `memory`로 전환되며, 메인 액티브 리스트(`App.tsx`)에서 즉시 사라지도록 로직 개선 완료. (테스트용 +10점 버튼 임시 할당)

## 8. 🎨 UI/UX 디테일 개선 및 반복 업무 미니 그래프 추가 (Added & Fixed)
- **실수 3: JellyBar 렌더링 색상 중복 배정 현상**
  - **이슈**: 기존 `taskId` 해싱 방식이 우연한 계산 결과로 인해 인접한 알약 바들이 같은 색상(예: 파란색 연속 2개)으로 렌더링되어 메인 화면의 색상이 겹치는 문제 발생.
  - **해결(Fixed)**: 색상 배정 방식을 `taskId` 해시에서 배열의 `index` 기반 순차 할당(`Math.max(0, index) % 6`)으로 변경하여 인접한 바끼리 절대 같은 색상이 나오지 않도록 보완 완료.
- **Task Input 입력창 X 버튼 이탈 버그 해결 (Fixed)**: 
  - **이슈**: Task 입력창 텍스트 필드가 길어지면서 flex 컨테이너를 강제로 늘려, 우측의 취소(X) 버튼이 하얀 배경 컨테이너 바깥으로 밀려나는(Overflow) 버그 발생.
  - **해결(Fixed)**: `.task-input-name`에 `min-width: 0;` 속성을 추가하여 flex item이 고유 너비 이하로 유연하게 줄어들 수 있도록 레이아웃 완벽 조정.
- **반복 업무 보관소(Recurring) 시간 시각화 그래프 추가 (Added)**: 
  - **기능**: 반복 업무 리스트의 시간 텍스트 하단에 메인 JellyBar와 유사한 느낌의 얇은 글래스모피즘 미니 바(Mini Graph)를 추가하여 시인성 확보.
  - **비율 로직**: 최대 60분을 100% 기준으로 삼아 `Math.min(100, (분/60) * 100)%` 공식을 적용. 시간이 늘어남에 따라 직관적으로 바가 채워지며, 60분을 초과해도 레이아웃 파괴 없이 안전하게 100% 꽉 찬 상태로 유지되도록 설계.
  - **디자인 심리학 (Color Psychology)**: 시간 압박감을 주는 붉은색 대신, 부교감신경을 자극하고 심리적 안정감과 평온함을 주는 소프트 민트 그린(Mint Green, `rgba(110, 231, 183, 0.65)`) 톤으로 변경하여 사용자 스트레스 완화.

## 9. 🔄 일일 도파민 게이지 자동 초기화 및 잉여 점수 로직 (Added)
- **오전 7시 자동 초기화 (`TaskStore.ts`)**:
  - `lastGaugeReset` 상태(Timestamp)를 도입하여, 마지막 초기화 시간과 현재 시간을 비교해 매일 오전 7시를 기준으로 게이지를 0으로 리셋하는 로직(`checkAndResetDailyGauge`) 구현.
  - 브라우저를 껐다 켜도 초기화 시점이 유지되도록 `persist` 미들웨어 파셜(partialize)에 포함.
- **잉여 점수 처리 로직 (`DailyGauge.tsx`)**:
  - `dailyGauge`가 100점을 초과하여 보상(슬롯머신)이 지급될 경우, 게이지가 처음부터 다시 채워지도록 100단위 잉여 점수(`dailyGauge % 100`)만 렌더링되게 연산 변경.
  - 마운트 시 및 점수 획득 시 리셋 체크 함수를 연동하여 오랫동안 켜둔 상태에서도 정확하게 동기화되도록 수정.

## 10. 📊 StatsWidget (도파민 기록장) UI/UX 고도화 (Fixed)
- **잔디 그래프 주간 고정 로직 변경**:
  - 기존의 롤링 방식(최근 7일)에서 '이번 주(일요일~토요일)' 고정 보기 방식으로 변경하여 사용자가 한 주의 달성도를 직관적으로 파악할 수 있도록 수정.
  - 미래 날짜(오늘 이후)에 대해서는 `opacity: 0.4`를 적용하여 아직 도래하지 않은 날짜임을 시각적으로 구분.
- **도파민 기록장 레이아웃 및 폰트 개선**:
  - 컴포넌트 내 텍스트 폰트를 서비스 테마에 맞게 일관성 있게 수정.
  - 콘텐츠 블록을 컴포넌트(이미지) 영역 중앙에 안정적으로 배치하여 전체적인 밸런스와 가독성을 대폭 상향.

## 11. 🗄️ 상단 헤더 액션 버튼 이동 및 레이아웃 정리 (Fixed)
- **버튼 위치 변경**: 
  - `App.tsx` 최상단 우측에 고정되어 있던 '기억 창고'와 '퇴근' 버튼을 우측 사이드 패널인 `StatsWidget.tsx` 하단('나의 도파민 기록' 밑)으로 이동.
- **스타일 및 접근성 개선**:
  - 사이드바 내부 너비에 맞춰 버튼이 1열(Column)로 정렬되도록 CSS 업데이트. 상단 헤더가 깔끔해지고 기능적 일관성(통계 및 히스토리 관리)을 사이드바로 통합.

## 12. 🧩 메인 화면 레이아웃 최적화 (DualModeToggle & DailyGauge 위치 조정) (Fixed)
- **이슈**: 우측 패널(`right-panel`) 내부가 세로(Column) 정렬로 되어 있어, 'ADHD 우리의 장점' 위젯 아래로 '일일 도파민 바'가 밀려나며 화면의 시각적 밸런스가 무너지는 현상 발생.
- **해결(Fixed)**:
  - 패널 레이아웃을 가로(Row) 배열로 변경하고 수직 중앙 정렬(`align-items: center`)을 적용.
  - '일일 도파민 바'가 패널 좌측에서 원래의 세로 길이를 확보하도록 기존 위치로 회귀.
  - 'ADHD 우리의 장점' 위젯을 도파민 바 우측 빈 공간 정중앙에 배치하여 UI 밸런스와 안정감을 대폭 개선함.

- **이슈**: 우측 패널을 가로 배열로 시도했으나, 원래의 세로 형태가 더 익숙하고 보기 편하다는 피드백 수렴.
- **해결(Fixed)**:

  - `App.tsx`에서 듀얼 모드 위젯('ADHD 우리의 장점')을 상단으로, 일일 도파민 게이지를 원래 위치인 그 아래로 배치하여 레이아웃을 깔끔하게 재정비함.
