# 🛠 ️TaskJelly 반복 업무 위젯 UI 가독성 수정 기록

## 📌 문제 상황
- `RecurringWidget` (반복 업무 보관소) UI를 추가했으나, 바탕색(밝은 테마)에 흰색 텍스트와 투명 배경이 지정되어 메인 화면에서 버튼과 내용이 보이지 않는('투명 인간') 문제가 발생함.

## 🔍 원인 파악 (다크모드 병)
- 개발 중 무의식적으로 다크 테마 방식(`color: white;`, `background: rgba(255, 255, 255, 0.2)`)을 인라인 및 CSS에 박아버림.
- 앱의 전체적인 테마는 아주 밝은 파스텔/글래스모피즘 톤이기 때문에 흰색 글씨와 뿌연 화이트톤 박스가 섞여 글씨가 보이지 않게 됨.

## 💡 해결 과정
1. **인라인 스타일 제거:** `RecurringWidget.tsx` 내의 `<span style={{color: 'white'}}>` 같이 강제 지정된 색상 코드를 걷어내고 클래스로 분리.
2. **테마 통일화 (`RecurringWidget.css` 교체):**
    - 메인 폰트 색상을 `#334155`(다크 네이비/그레이), 보조 폰트 `#64748b`로 수정하여 명도 대비를 확실히 줌.
    - 버튼 테마도 주변에 있는 "새 작업 추가"(`TaskInput`)의 파스텔톤 버튼 스타일에 맞추어 교체. (`rgba(255, 255, 255, 0.5)` 바탕, `#fb7185` 핑크빛 외곽선과 텍스트 포인트 활용)

## 🤔 교훈 및 향후 지침 (SKILL에 업데이트 완료)
- 새로운 컴포넌트를 UI에 덧붙일 때는 그냥 예뻐 보이는 코드를 짜지 말고 **주변 컴포넌트(`TaskInput.css` 등)의 컬러코드/테마(Color Palette, Design system)**를 파악 후 그대로 이식해야 자연스럽고 가독성이 확보된다.
