# 🛠 ️TaskJelly 반복 업무 위젯 UI 가독성 수정 기록

## 📌 문제 상황
- `RecurringWidget` (반복 업무 보관소) UI를 추가했으나, 바탕색(밝은 테마)에 흰색 텍스트와 투명 배경이 지정되어 메인 화면에서 버튼과 내용이 보이지 않는('투명 인간') 문제가 발생함.

## 🔍 원인 파악 (다크모드 병)
- 개발 중 무의식적으로 다크 테마 방식(`color: white;`, `background: rgba(255, 255, 255, 0.2)`)을 인라인 및 CSS에 박아버림.
- 앱의 전체적인 테마는 아주 밝은 파스텔/글래스모피즘 톤이기 때문에 흰색 글씨와 뿌연 화이트톤 박스가 섞여 글씨가 보이지 않게 됨.

## 💡 해결 과정
1. **인라인 스타일 제거:** `RecurringWidget.tsx` 내의 `<span style={{color: 'white'}}>` 같이 강제 지정된 색상 코드를 걷어내고 클래스로 분리.
2. **테마 통일화 (`RecurringWidget.css` 교체):**
    - 메인 폰트 색상을 `#334155`(다크 네이비/그레이), 보조 폰트 `#64748b`로 수정하여 명도 대비를 확실히 줌.
    - 버튼 테마도 주변에 있는 "새 작업 추가"(`TaskInput`)의 파스텔톤 버튼 스타일에 맞추어 교체. (`rgba(255, 255, 255, 0.5)` 바탕, `#fb7185` 핑크빛 외곽선과 텍스트 포인트 활용)

## 🤔 교훈 및 향후 지침 (SKILL에 업데이트 완료)
- 새로운 컴포넌트를 UI에 덧붙일 때는 그냥 예뻐 보이는 코드를 짜지 말고 **주변 컴포넌트(`TaskInput.css` 등)의 컬러코드/테마(Color Palette, Design system)**를 파악 후 그대로 이식해야 자연스럽고 가독성이 확보된다.

## 🚀 일일 도파민 게이지 & 레이아웃 최적화 (2026-04-24)

### 🎨 김대리 (UI/UX 디자인 총괄)
- **일일 도파민 게이지 디자인 개편 (`DailyGauge.css`)**
  - 게이지를 우측으로 배치하되 메인 콘텐츠와 더 가깝게 이동 (`left: calc(50% + 148px);`)
  - 차오르는 게이지 영역에 파스텔톤 그라데이션 및 하얀색 획(`3px solid rgba(255, 255, 255, 0.9)`)을 추가하여 차오르는 시각적 쾌감 극대화
  - "일일 도파민" 세로 텍스트를 `1.4rem`으로 키우고 `absolute`와 `flexbox`를 활용해 게이지 바 상하좌우 정중앙에 완벽 배치

### 👨‍💼 박상무 (비즈니스 로직 & 아키텍처 설계)
- **도파민 점수 산정 로직 기획 (`TimerStore.ts`)**
  - 기존 고정 10점 획득 방식에서 **작업 시간에 비례하는 보상 시스템**으로 개편하여 사용자 성취감(ADHD 맞춤) 증대 기획
  - `완료 점수 = 기본 10점 + (할 일의 설정 시간(분) × 2점)` 공식 수립

### 👩‍💻 유부장 (기능 구현 및 디버깅 전담)
- **점수 로직 코드 구현 (`TimerStore.ts`)**
  - 박상무의 기획에 따라 `Math.floor(state.totalTime / 60)`을 통해 초를 분 단위로 변환 후 타이머 완료 시 점수가 정확히 합산되도록 코드 적용
- **가로 스크롤바(슬라이드) 버그 완벽 디버깅 (`App.css`, `StatsWidget.css`)**
  - 리액트 최상위 컨테이너(`body, html, #root`)에 `overflow-x: hidden;`을 적용하여 1차 전역 레이아웃 이탈 방어
  - 브라우저 하단 스크롤바의 **진짜 원인**이 오른쪽 통계 패널(`.stats-sidebar`) 내부 달력 요소의 미세한 너비 삐져나옴 임을 끈질기게 추적하여 찾아냄. 패널 자체에 `overflow-x: hidden;`을 덧씌워 스크롤바 완전 박멸 성공
