## 1. 프로젝트 목표

성인용품 도매몰(리보스 등)에서 받은 상품을 쿠팡·지마켓·옥션·11번가에 등록할 때 발생하는 두 가지 문제를 자동화로 해결한다.

### 문제 A: 노출 이미지 → 마켓 반려
- 공급사 이미지에 여성 음부·가슴, 남성 성기 형태 제품(오나홀 등) 노출
- 네이버 등 마켓 등록 반려 사유
- → **포토샵으로 직접 처리하기로 결정** (자동 마스킹보다 빠름, 2분 내 완료)

### 문제 B: 아이템위너 경쟁 (쿠팡)
- 동일 공급사 → 여러 셀러가 같은 썸네일·상품명·상세이미지 사용
- 쿠팡이 동일 상품으로 묶어 아이템위너에서 떨어지면 가격경쟁 → 마진 손실
- → **상품명 SEO 차별화 + 상세페이지 부분 누끼로 독자 썸네일 제작**

## 6. 진행 계획 (다음 단계들)

- 결과를 SQLite + 엑셀로 저장

- `llama3.2-vision`으로 이미지 분류 (🟢🟡🔴)
- 🔴 이미지를 별도 폴더로 자동 분리 → 포토샵 작업 대상 명확화
- BiRefNet으로 상세페이지 부분 누끼 → 1000×1000 흰배경 썸네일 변환
- FastAPI에 엔드포인트 추가 (`POST /classify_image`, `POST /extract_subject`)

### 5단계: SQLite 이력 + pHash 중복 검사

- 공급사 동일 이미지 재사용 감지 → 캐싱으로 LLM 호출 절감

### 6단계 (보류): 마켓 자동 등록
- 단건 검수 업로드 방침이라 단건이면 수동이 더 안전 → 우선순위 낮음

## 10. 다음 작업자(Claude) 인수인계

> 미래의 Claude가 이 프로젝트를 이어받을 때 읽을 섹션.

### 핵심 컨텍스트
- 사장님(`세현 사장님`)께 **항상 존댓말** 사용
- 사장님은 **단건 검수 업로드** 방식 선호 (대량 자동화 X)

### 1·2단계에서 학습한 함정들 (다시 빠지지 말 것)
1. **qwen3 시리즈는 thinking 모델** — `/no_think` 지시자 + `format=json` 동시 사용 시 빈 응답
2. **VRAM 초과 = 추론 속도 20배 저하** — 모델 선택 시 VRAM 대비 80% 이하
3. **LLM 프롬프트만으로 카테고리 격리 안 됨** — 후처리 자동 교정 필수
4. **dedupe는 max_occurrences=2** — 1회는 SEO 손실, 3회는 스팸성
5. **PowerShell `Invoke-RestMethod` 한글 깨짐** — 인코딩 설정 후에도 깨짐, Swagger UI나 Python으로 우회 (D-019)
6. **네트워크 드라이브의 venv는 호환 불가** — Z:\에 만든 venv는 세컨 PC 전용, 메인 PC에선 별도 venv
