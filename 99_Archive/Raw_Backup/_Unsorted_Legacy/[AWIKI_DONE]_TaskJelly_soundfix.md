**이슈 요약:** 슬롯머신(DopamineWidget) 동작 시 스핀 사운드 및 결과창 팡파레 사운드 미출력 문제

## 1. 이슈 원인 분석 (What went wrong?)

제가 OpenGameArt에서 텐션을 올릴 카지노 스핀음(`slot-spin.mp3`)과 승리 팡파레(`slot-win.mp3`)를 바로 다운로드하여 에셋에 꽂아 넣으려 했습니다. 하지만 여기서 **치명적인 실수**가 있었습니다.

*   **실제 다운로드 된 파일의 정체:** 터미널 명령어로 직접 URL을 호출하여 다운로드하는 과정에서, OpenGameArt 서버의 보안 정책(혹은 URL 변경)으로 인해 오디오 파일이 아닌 **404 에러를 알리는 HTML 페이지 문서**가 MP3 확장자로 둔갑하여 저장되었습니다.
*   **브라우저 렌더링 에러:** 확장자는 `.mp3`였지만 속알맹이는 `<DOCTYPE html>`로 시작하는 텍스트 파일이었기 때문에, React의 `new Audio().play()`가 이 파일을 디코딩하지 못하고 내부적으로 `DOMException (NotSupportedError)`를 발생시키며 소리를 완전히 씹어버렸습니다.

## 2. 해결 과정 (How it was fixed)

1.  **오디오 파형 직접 합성 (By 유부장):** 외부 링크에 의존하다가 발생하는 다운로드 오류 및 404 차단 문제를 영구적으로 해결하기 위해, 유부장님이 파이썬(Python)의 `math.sin` 파형 계산을 통해 **주파수 대역을 직접 깎아 순수 `.wav` 파일로 구워냈습니다.**
    *   `slot-spin.wav`: 400Hz 대역을 기준으로 주파수가 빠르게 요동치는 긴장감 넘치는 스핀 파형 구현
    *   `slot-win.wav`: C5, E5, G5 코드를 활용하여 승리감(Tada!)을 주는 경쾌한 주파수 파형 구현
2.  **타임아웃 동기화:** 3초 뒤에 재생되는 타임아웃 딜레이(`setTimeout`)에 맞춰 사운드 객체를 선언하고, 올바른 `.wav` 파일 경로를 매핑하여 **브라우저의 재생 차단 이슈 없이 완벽한 타이밍에 타격감**을 주도록 수정 완료했습니다.

## 3. 향후 재발 방지 대책 (Next Steps)

*   **음원 파일 무결성 검증:** 외부에서 음원을 스크래핑하거나 가져올 때, 파일 용량(Bytes)과 헤더(Header) 정보를 반드시 확인하여 실제 오디오 파일 포맷인지 검증하는 단계를 거치겠습니다. 🎛️
*   **사운드 에러 핸들링:** `.play().catch(e => console.error(...))` 부분에서 단순히 로그만 찍는 것이 아니라, 오디오 소스가 깨졌을 경우 fallback(기본 비프음 등)을 띄우거나 명시적인 에러 알림이 오도록 오디오 파이프라인을 견고하게 짜겠습니다.
