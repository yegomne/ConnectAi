# 젤리 물리바 제작 (MatterJelly) 에러 수정 및 개선 일지

이 파일은 젤리바 컴포넌트를 제작하며 발생한 문제점을 기록하고 반복적인 실수를 방지하기 위한 지식 베이스입니다.

## 2026-04-21: 크기 불일치 및 쪼그라드는 현상
**이슈 1: 뷰박스(viewBox)와 물리 좌표계의 불일치로 인한 크기 문제**
- 증상: SVG 외곽선이 아주 작게 점처럼 렌더링되거나 비율이 깨짐. 기존 viewBox는 "0 0 600 300" 이었고, 생성된 모양은 260x80 이었음. 브라우저에서 화면 폭에 맞춰 렌더링할 때 심하게 축소되어 보임.
- 해결 방안: `viewBox="0 0 400 120"` 단위 환경으로 통일하고, 중심점을 `x=200, y=60`에 위치시킴. 물리 엔진은 이 해상도에 맞게 260x80 짜리 둥근 타원을 구성.

**이슈 2: 젤리가 쪼그라들어 모양을 잡지 못함**
- 증상: 표면 장력과 내부 지지력이 약해 형태를 유지하지 못하고 내부로 주저앉음 (Deflated).
- 해결 방안: 내부 압력을 모사하기 위해 다음과 같은 Constraint 구조를 강력하게 구성.
  1. **앵커(Anchor) 제약:** 중앙 `(200, 60)`에서 각 파티클까지의 거리를 계산해 강력하게 묶음.
  2. **슈퍼 크로스(Cross) 링크:** 입자들이 쪼그라들지 못하게 마주보는 반대편 입자들과 강한 `stiffness: 0.1`로 밀어내는(길이 유지) 링크 추가.

**이슈 3: 과도한 글래스모피즘 오버 엔지니어링**
- 증상: clipPath, foreignObject 등 불필요하게 복잡한 DOM 객체를 사용해 렌더링 에러 발생. 
- 해결 방안: 직관적이고 심플하게 수정. `<svg>` 자체를 CSS `backdrop-filter: blur(15px)` 로 처리하고(컨테이너 역할), 그 내부의 `<path>`는 `fill="rgba(255, 255, 255, 0.2)"` 만 적용해 심플하게 투명 블러 효과 구현.

**이슈 4: 텍스트 오버레이 정렬 문제**
- 증상: 젤리 안에서 글씨가 벗어남.
- 해결 방안: SVG 뒤에 깔리는 사각형 디자인을 `.jelly-bar-container` 등에서 투명(transparent)으로 모두 제거하고 순수하게 viewBox의 중심점 위에 absolute로 `z-index`를 올린 Flex 텍스트 영역을 고정시킴.
