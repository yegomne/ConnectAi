## 1. P - Purpose (목적 및 배경)
- **목적:** 메인 PC(작업 및 제어)와 세컨 PC(올라마 로컬 LLM 서버) 간 초고속 내부망(2.5Gbps)을 활용한 파일 공유 환경 구축.
- **배경:** 세컨 PC `C:\Users\pthem\Documents\adult_product_renamer` 폴더를 메인 PC에서 물리 하드디스크인 `Z:\` 드라이브처럼 인식시켜, 대용량 상품 데이터 및 이미지 소싱 자동화 스크립트가 지연 없이 원격으로 상호 작용할 수 있도록 함.

---

## 2. R - Resource (자원 환경)
- **메인 PC (Client):** - IP: `192.168.0.100` / OS 계정: `User`
- **세컨 PC (Server / Ollama):** - IP: `192.168.0.200` / OS 계정: `pthem` (마이크로소프트 연동 계정)
- **인프라:** 2.5Gbps 초고속 내부망 (공유기/스위치 바운더리 내부)

---

## 3. A - Architecture & Analysis (구조 및 원인 분석)
### [인프라 통신 구조]
- **올라마(Ollama) 통신:** 세컨 PC(`200`)의 `11434` 포트를 개방하여 메인 PC와 API 서빙 통신 (포트 제어 라인).
- **파일 공유(SMB) 통신:** 세컨 PC의 로컬 스토리지 권한을 개방하여 메인 PC가 `Z:` 드라이브로 마운트 (파일 시스템 라인).

### [핵심 트러블슈팅 및 원인 분석]
1. **오류 86 (지정된 네트워크 암호가 맞지 않습니다):** - *원인:* 세컨 PC의 계정이 마이크로소프트 온라인 연동 계정(`pthem@hotmail.com`) 및 Windows Hello(PIN 번호) 보안 인증을 사용하여 일반적인 SMB `net use` 방식의 암호 주입을 거부함.
2. **오류 8646 (지정된 계정에 대한 권한이 없음):**
   - *원인:* 온라인 공급자 계정 특성상 윈도우 파워쉘에서 `net user` 명령어로 비밀번호 강제 제어 및 할당이 불가능함.
3. **오류 67 (네트워크 이름을 찾을 수 없습니다):**
   - *원인:* 메인 PC와 세컨 PC의 IP 방향성 혼선 및 윈도우 계정 폴더명(`pthem`)과 실제 시스템 등록 ID(`User`) 불일치로 인한 경로 미발견.
4. **Permission Denied (액세스가 거부되었습니다):**
   - *원인:* 네트워크 망은 연결되었으나 세컨 PC 윈도우 파일 시스템(NTFS) 자체의 보안 탭에서 네트워크 접근 계정에 대한 권한(ACL)을 차단함.
