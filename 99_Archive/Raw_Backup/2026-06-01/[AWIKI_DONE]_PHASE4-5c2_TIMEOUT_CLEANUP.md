## 1. 제품 완성 팝업 (4-5c-3 설계 확정 — 사장님 설계)

[제품 완성] 누르면 바로 다음 안 가고 **완성 확인 팝업** = 마켓 등록 작업대:
- **상품명 클릭 → 클립보드 복사** ("복사됨" 표시) → 마켓에 붙여넣기.
- **상세페이지/썸네일 이미지 작게 표시 + 외부로 드래그**
  · QDrag + mimeData.setUrls([file://]) → 탐색기 폴더/브라우저 마켓에 드롭.
  · "프로그램 → 외부" 방향 (탐색기 확실, 마켓 브라우저는 받으면).
- [완성 폴더 열기] / [등록 완료].
- 완성품을 바로 마켓에 올리고 넘어가는 흐름.

## 2. 실사용 발견 3가지 이슈 + 해결

실제로 C-892(구강/음부 실리콘 상세컷)로 작업하다 발견.

### 이슈 1: 큰(붉은) 이미지에서 "응답 없음" 멈춤

**사장님 진단 (정확)**:
- 이미지 크기 문제 아님 (다른 건 잘 됨).
- 그 이미지의 절반이 **음부 내부를 표현한 붉은색** →
  NudeNet 검수가 거기서 **과부하** → 뻗음(응답 없음).

**해결: 검수 타임아웃 10초**
- 검수(image_screener.screen)를 백그라운드(QThread) + 타임아웃 10초.
- 10초 안에 안 끝나면 → 검수 포기 + "검수 시간 초과 — 검수 없이 진행".
- UI 멈춤 방지가 핵심. 검수 없이 사장님 수동 가리기 진행.
- 카테고리 위험도는 유지 (실리콘 = HIGH 카테고리라 최소 안전망 작동).
- 근본 원인은 CPU 처리 → 나중에 GPU 살리면 빨라짐(REF_GPU_CUDA_FIX.md).

### 이슈 2: [등록 완료] 후 다음 제품으로 안 넘어감

**증상**: 완성 팝업에서 [등록 완료] 눌러도 화면 그대로, 새 제품 안 됨.

**해결 (사장님 선택 A)**: [등록 완료] →
1. 완성 팝업 닫기
2. 작업 화면 초기화 (상품명/카테고리/키워드/캔버스/검수 리셋)
3. **[+ 새 제품] 팝업 자동으로 띄움** → 바로 다음 제품 시작.
- 등록 완료 = 마무리 + 다음 제품 준비 한 번에.

### 이슈 3: 사용 안 할 파일 처리

**사장님 판단**: 등록 완료 시 **완전 정리** (원본까지 삭제).
- 근거: **리보스에 원본이 있고, 리보스에서 내린 물건이면 사진이 불필요.**
  → 로컬에 원본/중간산물 남길 이유 없음. 완성품만.

**해결: 등록 완료 시 완전 정리**
- 남길 것 (완성품만, data/작업/[원본상품명]/):
  · 상품명.txt, 상세페이지.jpg, 썸네일_N.jpg, 정보.json
- 지울 것 (이 제품 관련 전부):
  · 원본 드래그 이미지 (K-574-860.jpg 등) ← **원본도 삭제**
  · processed/masked, cutout, merged (이 제품 것)
  · downloads (이 제품 임시)
- **절대 안 지움: data/training_images/** (학습 데이터, corrected 등 영구 보존)
- ★ 안전장치 (원본까지 지우므로 필수):
  · **완성품 저장 100% 확인 후에만 삭제** (저장 실패 시 삭제 안 함)
  · 이 제품 관련 파일만 식별 삭제 (다른 제품 안 건드림)
  · try/except — 삭제 실패해도 완성품 안전, 저장이 우선
