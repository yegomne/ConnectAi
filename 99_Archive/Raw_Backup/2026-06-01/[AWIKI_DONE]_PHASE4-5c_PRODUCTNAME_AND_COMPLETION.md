## 3. 4-5c-2a' 위험도 강화학습 [코드 넣고 믿고 넘어감]

- 사장님 아이디어: 위험도도 고정이 아니라 **강화학습으로** 진화.
- 신호 = **노출 가리기 한 횟수** (사장님이 가리면 = 위험).
- SQLite `category_risk_stats(category_key, work_count, covered_count)`.
- 가린 비율로 위험도 자동 조정:
  · ↑(올림) 빠르게 — 안전 우선
  · ↓(내림) 보수적 — 노출 제품을 LOW로 잘못 내리면 마켓 반려.
    HIGH 초기값은 work_count 많고 비율 매우 낮을 때만 단계적 하향.
- 초기값 + 학습값 결합: work_count < 5면 초기값 신뢰, 쌓이면 학습값.
- **테스트 어려움** (여러 번 가려야 학습 보임) → 키워드 강화학습과 동일 구조라
  **믿고 넘어감** (Q: 다 추천대로, B 선택). 통계 누적 로직만.

## 8. ★ 폴더명 적용 버그 (2단계로 수정)

### 증상
- 새 제품 [만들기] 시: 이미지가 `data/downloads/`에만 저장, 원본상품명 폴더 안 만들어짐.
- 폴더명 미리보기는 나오는데 실제 적용 안 됨.

### 원인
- [만들기]가 다운로드만 하고 safe_folder_name으로 폴더 생성/이동을 안 함.

### 수정 1: 폴더 생성 + 이미지 이동
- [만들기] → `data/작업/[safe_folder_name(원본상품명)]/` 생성 → downloads 이미지 이동.
- safe_folder_name: 윈도우 금지문자(\ / : * ? " < > |) 제거, 끝 공백/마침표 제거,
  길이 제한(50), 빈값이면 기본명, 중복이면 _2.
- 결과: **디스크에 원본상품명 폴더 생성 + 이미지 이동 OK** ✓

### 수정 2: 목록 표시명 불일치 (절반 남았던 것)
- 디스크 폴더는 원본상품명인데, **프로그램 좌측 그룹 목록은 여전히 "k-574"**(파일코드).
- 수정: 그룹 헤더 표시를 **원본상품명**으로.
  · 내부 그룹키 = 파일명 기반(k-574, 묶기/정렬용) 유지
  · 표시 이름 = 원본상품명
- 결과: 폴더명 = 목록 표시명 = 원본상품명 일치 ✓

- **상품명 클릭 → 클립보드 복사** ("복사됨" 표시) → 마켓에 붙여넣기.
- **이미지(상세페이지/썸네일) 작게 표시 + 드래그로 외부에**
  · QDrag + mimeData.setUrls([file://]) → 탐색기 폴더/브라우저 마켓에 드롭.
  · "프로그램 → 외부" 방향 드래그 (탐색기 확실, 마켓 브라우저는 받으면).
- [완성 폴더 열기] (os.startfile), [등록 완료] → 다음 제품으로.
- 드래그 목적지: C(둘 다 — 탐색기/마켓) 권장.

→ 완성품을 바로 마켓에 올리고 넘어가는 흐름.

## 11. 전체 워크플로우 (완성형)

```
[+ 새 제품] → 원본상품명 입력 + 리보스 이미지 드래그(URL 다운로드)
  → 원본상품명 폴더 생성 + 자동 그룹
  → 카테고리 선택 (검수 위험도 연동 + 위험도 학습)
  → 상품명 (5후보 + 6번째 조립, 키워드 학습)
  → 이미지 편집 (검수 → 노출/광고 가리기 → 누끼/썸네일)
  → 제품 합치기 (세로 결합)
  → [제품 완성] → 폴더 정리 + DB + 완성 팝업
       팝업: 상품명 복사, 이미지 드래그 → 마켓 등록
       [등록 완료] → 다음 제품
[처음으로] → 갈아타기
```

해결한 두 문제:
- 마켓 반려 → 검수 + 노출 가리기
- 아이템위너 → 독자 상품명(SEO) + 독자 썸네일(누끼)
