## 1. 검수 멈춤 — 정확한 재진단 (오해 정정 과정)

### 증상 (사장님 최종 설명, 정확)
C-892(붉은 음부 내부 비중 큰 실리콘 상세컷)에서:
1. **이미지가 나오면 "응답 없음" 한 번** (조금 있다 풀림) — 이미지 로드/검수.
2. 풀린 후 **가리기 작업하려고 클릭만 하면 또 멈춤**.

### 오해와 정정 (기록용)
- 1차 오해: "이미지가 커서" → 사장님: 크기 아님, 붉은 음부 내부에 검수 과부하.
- 2차 오해: "가리기 합성/저장에서 멈춤" → 사장님: 클릭만 해도 멈춤.
- 최종: **두 번 멈춤** (이미지 나올 때 1회 + 클릭할 때마다).

### 유력 원인
- **클릭/박스 조작마다 검수(NudeNet)가 재실행되는 것**으로 의심.
- C-892는 검수가 무거워 → 조작할 때마다 검수 돌면 매번 멈춤.
- 앞서 넣은 검수 타임아웃이 오히려 클릭마다 트리거됐을 가능성.

### 해결 방향 (프롬프트로 준비, 적용 보류)
- **검수는 이미지당 1회만** (선택 시). 클릭/드래그/박스조작/줌/렌더 때 재실행 금지.
  검수 결과 캐시 (이미지별 보관).
- 이벤트 핸들러(mousePress/Move/paint)는 가볍게 (좌표/그리기만).
- 첫 검수도 백그라운드(별도 프로세스) + 10초 타임아웃 → UI 안 멈춤.
- (옵션) HIGH 카테고리는 검수 생략 — 실리콘 음부는 NudeNet이 어차피 못 잡고
  카테고리 HIGH로 "확인 필요" 뜨므로.

### 사장님 결정
- "우선 나만 쓰니까 이대로 써본다." → 검수 1회화 수정은 보류, 필요 시 적용.
- (멈춤이 작업 흐름을 자주 끊으면 위 프롬프트 적용 권장. 근본은 GPU로 빨라지면 완화.)

## 2. 실행환경 — 배치 파일 + 바탕화면 바로가기

### 배포 방식 결정 (혼자 사용 = exe 불필요)
- 혼자 쓰고 계속 개선 중 → **exe 안 만듦**.
  · exe 단점: 수정마다 재빌드, PySide6+NudeNet+BiRefNet 묶으면 거대/까다로움, 디버깅 불리.
- **배치 파일(.bat) + 바로가기**로 더블클릭 실행. 수정 즉시 반영.
- exe는 나중에 도구 완전 안정화 후 고려.

### 만든 것
- `E:\성인용품 등록 반자동화\리보스자동.bat`:
  ```bat
  @echo off
  chcp 65001 >nul
  cd /d "E:\성인용품 등록 반자동화"
  call .venv\Scripts\activate
  cd adult_product_renamer
  python desktop\canvas_test.py
  ```
  · chcp 65001 = 한글 깨짐 방지. 에러 시 pause 고려.
- **바탕화면 바로가기 "리보스자동"** (.lnk):
  · TargetPath = 리보스자동.bat
  · IconLocation = E:\성인용품 등록 반자동화\app_icon.ico
  · win32com(pywin32)로 생성 (없으면 pip install pywin32 --break-system-packages)
- 더블클릭 → 프로그램(GUI) 실행. 검은 콘솔창은 같이 뜸(에러 보이게 일반 콘솔 유지,
  거슬리면 WindowStyle 최소화/숨김 가능).

### 주의
- 이 바로가기는 **메인 PC 프로그램만** 실행.
- **상품명용 세컨 PC 서버는 따로 켜야** 함 (192.168.0.200:8000, venv 활성화).
  · 필요 시 세컨 PC에도 서버 자동시작 배치 별도 구성 가능 (나중).
