## 1. 프로젝트 목표

성인용품 도매몰(리보스 등)에서 받은 상품을 쿠팡·지마켓·옥션·11번가에 등록할 때 발생하는 두 가지 문제를 자동화로 해결한다.

### 문제 A: 노출 이미지 → 마켓 반려
- 공급사 이미지에 여성 음부·가슴, 남성 성기 형태 제품(오나홀 등) 노출
- 네이버 등 마켓 등록 반려 사유
- → **포토샵으로 직접 처리하기로 결정** (자동 마스킹보다 빠름, 2분 내 완료)

### 문제 B: 아이템위너 경쟁 (쿠팡)
- 동일 공급사 → 여러 셀러가 같은 썸네일·상품명·상세이미지 사용
- 쿠팡이 동일 상품으로 묶어 아이템위너에서 떨어지면 가격경쟁 → 마진 손실
- → **상품명 SEO 차별화 + 상세페이지 부분 누끼로 독자 썸네일 제작**

- 기능: 공급사 원본 상품명 → 쿠팡/지마켓/옥션/11번가 통과 기준 SEO 상품명 5개 후보 자동 생성

- 카테고리 오염 자동 교정 작동 확인
- LLM 자체 오타는 사장님이 5개 후보 중 선택 시 거름 (수용 가능)

## 5. 진행 계획 (다음 단계들)

### 2단계: FastAPI 서버화 ← **다음 작업**
- 현재 CLI 도구를 HTTP API로 변환
- 엔드포인트: `POST /rename`, `GET /health`
- 세컨 PC에서 상시 구동
- keep_alive=30m으로 모델 로딩 비용 최소화

### 3단계: 단건 검수 GUI
- Streamlit 권장 (사장님 결정 대기 중)
- 한 화면에 원본 상품명 → 5개 후보 버튼 → 클릭 선택 → 다음 상품
- 결과를 SQLite + 엑셀로 저장

### 4단계: NSFW 스크리너 + 누끼 추출 통합
- llama3.2-vision으로 이미지 분류 (🟢🟡🔴)
- 🔴 이미지를 별도 폴더로 자동 분리 → 포토샵 작업 대상 명확화
- BiRefNet으로 상세페이지 부분 누끼 → 1000×1000 흰배경 썸네일 변환

### 5단계: SQLite 이력 + pHash 중복 검사
- 처리한 상품 이력 DB
- 공급사 동일 이미지 재사용 감지 → 캐싱으로 LLM 호출 절감

### 6단계 (보류): 마켓 자동 등록
- Playwright로 쿠팡 윙·스마트스토어 자동 입력
- 사장님 검수 업로드 방침이라 단건이면 수동이 더 안전 → 우선순위 낮음

세컨 PC `ollama list` 기준:

| 모델 | 크기 | 1단계 사용 여부 | 향후 용도 |
|---|---|---|---|
| **qwen3:14b** | 9GB | ✅ 메인 사용 | 상품명 리네이밍 (확정) |
| qwen3:8b | 5.2GB | 검증 후 탈락 | 백업/오프라인용 |
| qwen3-coder:30b | - | 미사용 | 안티그래비티 로컬 폴백 |
| qwen3.6:27b | 17GB | ❌ VRAM 초과 (eval 3 tok/s) | **삭제 권장** |
| qwen3.6:35b-a3b | 23GB | ❌ VRAM 초과 | **삭제 권장** |
| llama3.2-vision | 7.8GB | 미사용 | **4단계 NSFW 스크리너** |

### 모델 선정 결론
- **9~12GB 모델이 RTX 5060 Ti 16GB의 골든존**
- 13GB 초과 모델은 VRAM 부족으로 일부 CPU 처리 → 3 tok/s 참사
- qwen3:14b가 60+ tok/s로 정확히 골든존 점유
