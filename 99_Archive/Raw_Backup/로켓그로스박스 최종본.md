# AI 역방향 박스 계산기 (로켓그로스박스 최종본)

해당 문서는 `로켓그로스박스 최종본.html` 파일에 구현된 **AI 역방향 박스 계산기** 싱글 페이지 애플리케이션(SPA)의 구조와 주요 기능을 분석한 명세서입니다.

## 1. 개요
* **프로젝트명:** AI 역방향 박스 계산기 v2.1 (Regular Box & Final Rules)
* **형태:** 단일 HTML 파일 (CDN을 통해 라이브러리를 로드하여 서버 없이 동작)
* **목적:** 작은 상자(상품)들의 크기와 수량을 입력하면 이를 모두 담을 수 있는 '큰 박스'의 최적 사이즈를 역산하거나, 고정된 '큰 박스' 내에 작은 상자들이 어떻게 적재되는지 3D로 시뮬레이션하는 웹 애플리케이션입니다.
* **주요 기술 스택:**
  * **프레임워크:** React 18, ReactDOM
  * **스타일링:** Tailwind CSS, 커스텀 CSS (애니메이션, 스크롤바)
  * **트랜스파일러:** Babel (브라우저 내 JSX 실시간 변환)
  * **렌더링 방식:** SVG를 활용한 커스텀 3D 투영 알고리즘 (Three.js 등 외부 3D 라이브러리 없이 자체 구현)

## 2. 핵심 기능

### 2.1. 3D 적재 시뮬레이션 및 시각화
* **커스텀 3D 렌더러:** 캔버스(SVG) 상에 박스가 쌓이는 형태를 3D로 투영하여 실시간으로 렌더링합니다.
* **인터랙티브 뷰포트:** 마우스 드래그 및 터치(핀치 줌) 인터페이스를 지원하여 사용자가 자유롭게 시점을 회전하고 확대/축소할 수 있습니다.
* **스마트 가이드 및 경고 시스템:** 적재된 부피가 설정된 큰 박스의 내경을 초과(오버플로우)할 경우, 붉은색 테두리와 펄스 애니메이션 경고를 표시합니다.
* **자동 회전 (Auto-Rotate):** 적재 결과를 다각도로 확인할 수 있도록 시점이 자동 회전하는 기능을 제공합니다.

### 2.2. 두 가지 핵심 계산 모드
* **수동 모드 (고정 모드):** 사용자가 '큰 박스'의 크기(우체국 박스 규격 등)를 직접 지정하고, 그 안에 작은 박스들이 어떻게 빈틈없이 들어가는지 확인합니다.
* **자동 모드 (추천 모드):** 입력된 작은 박스들의 총 부피를 합산하고 알고리즘을 통해 이들을 모두 담을 수 있는 최적의 '추천 큰 박스 사이즈'를 자동으로 계산해 줍니다.

### 2.3. 혼적 모드 지원
* 단일 종류의 박스뿐만 아니라, **최대 4종류**의 각기 다른 사이즈를 가진 작은 박스들을 섞어서 적재하는 '혼적 모드'를 지원하여 실무 활용도를 높였습니다.

### 2.4. 패킹 알고리즘 (Reverse Packing Algorithm)
* 입력된 박스들을 체적(부피)이 큰 순서대로 정렬한 뒤, 각 상자를 6방향(회전)으로 돌려가며 가장 최적의 위치(Z축 바닥부터 차곡차곡)를 찾는 3D 빈 패킹(Bin Packing) 알고리즘이 적용되어 있습니다.

## 3. 프리셋 및 데이터 지속성
브라우저의 `localStorage`를 활용하여 사용자의 작업 상태와 프리셋을 자동으로 저장합니다.
* **큰 박스 프리셋 (`aiBoxCalc.regularBoxes.v1`):** 우체국 박스 1~4호와 같은 규격 박스 사이즈를 저장하고 관리합니다.
* **작은 박스 프리셋 (`aiBoxCalc.smallBoxSizes.v1`):** 자주 사용하는 상품의 치수(예: 기본 10×10×10)를 저장해두고 원클릭으로 불러올 수 있습니다.
* **UI 상태 저장 (`aiBoxCalc.uiState.v1`):** 마지막으로 입력했던 박스 수, 모드, 카메라 줌 및 회전 상태를 저장하여 새로고침 후에도 작업을 바로 이어서 할 수 있습니다.

## 4. UI 구성 요소 요약
1. **상단 컨트롤:** 입력 초기화, 공장 초기화 등 글로벌 상태 리셋 기능.
2. **좌측 패널 (제어부):** 
   * '큰 박스'의 규격 설정 (수동/자동 토글, 프리셋 관리).
   * '작은 박스'의 수량 및 치수 입력 제어 (혼적 모드 토글, 최대 4개 슬롯 제공).
3. **우측 패널 (뷰어부):**
   * 3D 적재 시뮬레이션 캔버스.
   * X, Y, Z 축에 대한 실제 적재 vs 제한 내경 수치 실시간 비교 모니터링 창.

## 5. 최근 업데이트 및 기능 개선 사항

### 5.1. Extreme Point(EP) 휴리스틱 3D 패킹 적용 (유부장)
* 기존의 단순 모서리(3방향) 후보지 생성 알고리즘의 한계를 극복하기 위해 **Extreme Point(EP) 3D 빈 패킹 알고리즘**을 전면 도입했습니다.
* 새 박스가 적재될 때 기존 빈 공간 후보지들과 겹치는지를 수학적으로 검사하고, 겹칠 경우 그 공간을 감싸는 모든 틈새 좌표들을 새롭게 역산하여 추가합니다. 
* 이를 통해 크기가 다른 상자들을 혼적할 때 생기는 내부의 미세한 틈새(Gap)까지 놓치지 않고 정확히 채워 넣을 수 있게 되었습니다.

### 5.2. 적재 로직 고도화 및 '꽉 채우기' 기능 추가 (사용자)
* **`packItems`의 `skipOnFail` 옵션 추가:** 기존에는 박스가 하나라도 들어가지 않으면 즉시 실패(`success: false`)를 반환했으나, `skipOnFail=true` 모드를 통해 들어가지 않는 큰 박스는 건너뛰고, 남은 작은 박스가 틈새를 마저 채울 수 있도록 로직이 개선되었습니다.
* **`handleMaxFill` 함수 구현:** 현재 컨테이너 부피를 기반으로 각 박스 종류별로 최대 가능 수량(최대 1000개 상한) 풀(Pool)을 생성한 후, 부피 내림차순 정렬과 `skipOnFail` 옵션을 활용하여 물리적인 빈 공간이 없어질 때까지 꽉 채워 넣는 기능입니다.
* **UI 버튼 분리 및 고도화:** 작은 박스 설정 영역에 기존의 비율 기반 최적화 버튼 외에, 비율과 무관하게 틈새를 극악으로 채워 넣는 **`수량 최대화 (꽉 채우기)`** 인디고(Indigo) 색상 버튼을 추가했습니다. 크기 차이가 심한 박스들을 섞을 때 잔여 공간 활용을 극대화합니다.

### 5.3. 상태 관리(State) 호이스팅 및 의존성 배열(deps) 버그 수정
* **원인:** React의 `useEffect`(UI 상태 저장, 작은 박스 프리셋 저장) 로직이 의존성으로 참조하는 `useState` 선언부보다 상단에 위치해 있었습니다. 이로 인해 Babel 변환 과정에서 호이스팅이 발생할 때, effect 초기 호출 시점에 상태값이 `undefined`로 평가되었습니다. 그 결과 deps 배열이 `[undefined]`로 등록되어 React가 변경을 감지하지 못하고 마운트 시 1회만 실행되었습니다.
* **증상:** 
  * 작은 박스 사이즈 프리셋을 추가해도 화면에만 반영될 뿐, 실제 저장이 되지 않아 새로고침 시 사라지는 현상이 발생했습니다.
  * UI 상태(`boxCount`, `smallBoxes`, 줌, 회전 등) 저장에도 동일한 버그가 있어, 과거 "혼적 모드에서 박스 종류 슬라이더가 안 바뀌는 것처럼 보였던" 현상의 주요 원인으로 작용했습니다.
* **수정 내역:** 문제가 된 두 개의 `useEffect`를 모든 `useState` 선언 위치보다 아래로 이동시켰습니다. 이제 deps 배열이 실제 상태값을 올바르게 참조하게 되어, 변경이 발생할 때마다 정상적으로 `localStorage`에 동기화(저장)됩니다. (`regularBoxes` 관련 effect는 기존에도 선언부 아래에 있어 영향을 받지 않으므로 유지했습니다.)
